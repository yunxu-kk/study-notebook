## 影响范围
- 所有使用批量导出功能的用户
- 可能导致导出空数据或全量数据，造成混淆
- 不符合用户交互预期

## 可能的原因分析
前端批量导出按钮的点击事件处理函数中，缺少对选中数据行数的校验逻辑。应该在执行导出操作前检查是否有选中的记录，如果没有则弹出提示并终止操作。

## 建议修复方案
在批量导出按钮的点击事件处理函数中添加校验逻辑：
```javascript
handleBatchExport() {
  if (this.selectedRows.length === 0) {
    this.$modal.msgWarning('请选择要导出的数据');
    return;
  }
  // 执行导出操作
}
```
